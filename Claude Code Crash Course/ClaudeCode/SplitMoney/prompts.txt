Data models (from PRD.md):

Group:
{
  "id": "uuid string",
  "name": "string",
  "members": [{"id": "uuid", "name": "string"}],
  "created_at": "ISO datetime"
}

Expense:
{
  "id": "uuid string",
  "group_id": "string",
  "description": "string",
  "amount": "float (in ₹)",
  "paid_by": "member id string",
  "split_among": ["member id strings"],
  "date": "ISO date string",
  "created_at": "ISO datetime"
}

API endpoints to build:
POST   /groups                              — create a group (name only)
GET    /groups                              — list all groups  
GET    /groups/{group_id}                   — get group + members
POST   /groups/{group_id}/members           — add member (name only)
POST   /groups/{group_id}/expenses          — add expense
GET    /groups/{group_id}/expenses          — list all expenses for group
DELETE /groups/{group_id}/expenses/{exp_id} — remove an expense
GET    /groups/{group_id}/settlement        — calculate who owes whom

Settlement calculation logic:
1. For each expense: paid_by gets +amount, each member in split_among gets -(amount/len(split_among))
2. Calculate net balance per member (positive = they are owed money, negative = they owe money)  
3. Minimise transactions: pair highest positive balance with highest negative balance
4. Return list of: { "from": member_name, "to": member_name, "amount": float }

Return all amounts rounded to 2 decimal places.
Return HTTP 404 with clear message for missing group or expense IDs.
Add startup event to create backend/data/ directory if it doesn't exist.

a detailed expense analysis for any group.

Frontmatter:
  name: settlement-calculator
  description: Deep analysis of expense patterns in a SplitEasy group. Use when you want more than just the settlement — who paid the most overall, fairness score, patterns across multiple expenses, suggestions for simpler splitting next time.
  tools: Read, Glob
  disallowedTools: Write, Edit, Bash
  model: claude-haiku-4-5
  color: green

System prompt:
You are a personal finance analyst specialising in shared expense fairness.
You have read-only access to the SplitEasy data files.

When given a group name or group ID, read:
- backend/data/groups.json (find the group and its members)
- backend/data/expenses.json (find all expenses for that group)

Produce a detailed analysis:

1. EXPENSE SUMMARY
   - Total group spending
   - Per-member: total paid, total owed, net position (+ means owed, - means owes)

2. FAIRNESS SCORE
   - Calculate coefficient of variation in contributions
   - Simple label: "Fairly split" / "Somewhat uneven" / "Very uneven"

3. TOP PAYER
   - Who paid the most across all expenses
   - What percentage of total they covered

4. SUGGESTED SIMPLIFICATION
   - If the group tends to split everything equally, suggest:
     "This group splits everything equally — consider using the equal split by default"
   - If one person always pays, suggest they collect cash at the start

Return a concise, friendly analysis. Use ₹ symbol. Keep it under 200 words.
